サービス概要

地域おこし協力隊員が毎日の業務を音声で記録するだけで、日報が自動的に整理・記録されるLINE Botサービス。 手入力・PCでの入力作業を一切不要にし、移動中・帰宅途中など隙間時間での日報提出を実現する。

項目内容
サービス名音声日報サービス（nichishi-bot）
対象ユーザー地域おこし協力隊員（パインヒルズ）
主な利用シーン帰宅途中・移動中・業務終了直後
利用端末スマートフォン（LINEアプリ）
記録先個人のGoogleスプレッドシート
通知先Slack（共有事項・管理通知）

コンセプト

データフロー

フロー経路
音声送信 LINE → Cloud Run → Gemini API → 確認メッセージ → LINE
日報記録 確認OK → Cloud Run → Google Sheets（テンプレートコピー＋書き込み）
Slack通知 Cloud Run → Slack Webhook（共有事項・登録通知）
スプシ設定通知 マスタースプシ編集 → Apps Script → Slack Webhook
新規登録 LIFF → Cloud Run（/register） → マスタースプシ → Slack通知

ユーザーフロー

初回登録フロー

1
友だち追加 LINE公式アカウントを友だち追加。ウェルカムメッセージが自動送信される。
2
リッチメニュー「初めての方はこちら」をタップ LINE内ブラウザ（LIFF）で登録フォームが開く。
3
名前・メールアドレスを入力して送信 LINE IDと紐付けてマスタースプシに記録される。Slackに登録通知が届く。
4
管理者がスプレッドシートを準備・Slackで案内 準備完了後、Slackで通知。以降は音声日報が利用可能。

日報送信フロー（日常利用）

1
マイクボタンを長押しして音声を送信 今日の業務内容を話す。時間帯・活動内容・共有事項など。
2
「受け取りました！整理中...」がすぐ返信される 10〜30秒の処理中も届いたことが確認できる。
3
構造化された日報内容が確認メッセージで届く 日付・活動内容・共有事項がフォーマット済みで表示される。
4a
「✅ はい」をタップ スプレッドシートに記録。共有事項があればSlackにも送信。「X月X日の日報を記録しました」と完了通知。
4b
「✏️ 修正する」をタップ 修正内容を音声またはテキストで送ると、再度AIが整理して確認画面を表示。
4

管理者フロー

新規メンバー受け入れ手順

1
Slackに登録通知が届く 「新規メンバーが登録しました：名前・メールアドレス」
2
日報用スプレッドシートを作成 テンプレートをコピーして個人用スプシを準備する。
3
マスタースプシのC列にスプシURLを貼り付け URLを貼るだけでIDが自動抽出される（Apps Script）。Slackに設定完了通知が届く。
4
Slackでメンバーに利用開始を案内 以降はメンバーが自律的に音声で日報を送信できる。

機能仕様

LINE Bot機能一覧

機能トリガー動作
ウェルカムメッセージ 友だち追加 サービス説明＋リッチメニュー案内を自動送信
未登録案内 テキスト送信（未登録ユーザー） 「初めての方はこちら」へ誘導
スプシ準備待ち案内 音声・テキスト送信（スプシ未設定） 「準備中です、もうしばらく」と返信
音声受付フィードバック 音声送信 「受け取りました！整理中...」を即時返信
音声→日報変換 音声送信（登録済み） Gemini APIで構造化。確認メッセージをpush送信
日報確認 「✅ はい」タップ スプシに記録＋Slack通知（共有事項あれば）
修正フロー 「✏️ 修正する」タップ 修正内容を受付→再構造化→再確認
エラーハンドリング Geminiタイムアウト・空レスポンス ユーザー向けに状況を説明したメッセージを返信

データ設計

マスタースプレッドシート（メンバー管理）

シート名：メンバー
列内容設定者備考
A列LINE ユーザーID自動（LIFF）Uから始まる文字列
B列氏名自動（LIFF）フルネーム
C列日報スプシID管理者URLを貼ると自動変換
D列メールアドレス自動（LIFF）連絡先として保持

日報スプレッドシート（個人）

シート：●月●日（テンプレート）→ 日次コピー（例：4月4日）
セル内容自動入力
C3日付○（令和〇年〇月〇日）
C6氏名○（マスタースプシから取得）
A10〜A16時間帯○（音声から抽出）
C10〜C16活動内容（最大7件）○（音声から抽出）
B17共有事項（Slack共有内容）○（音声から抽出）
同日に2回以上送信した場合、テンプレートの再コピーはスキップされ、既存シートへの上書きとなります。

インメモリ状態管理（サーバー）

変数型用途
pendingdict確認待ち中の構造化テキスト（user_id → text）
pending_correctionset修正モード中のユーザーID

実装状況

✅
音声→日報変換・スプレッドシート記録
Gemini APIによる構造化・テンプレートコピー・セル書き込みまで完全自動
✅
LIFF登録フォーム
二重登録防止・LINE ID自動取得・GitHub Pagesにホスティング済み
✅
音声処理中の即時フィードバック
受信直後に「整理中...」を返信。結果はpush通知で届く
✅
修正フロー
「修正する」タップ後、テキスト・音声どちらでも修正可能
✅
エラーハンドリング
タイムアウト・空レスポンス時にユーザーフレンドリーなメッセージを返信
✅
マスタースプシURL自動変換（Apps Script）
C列にURLを貼るだけでIDが自動抽出される
⏳
Slack通知（実装済み・URL設定待ち）
SLACK_WEBHOOK_URLを環境変数に設定するだけで即時有効化
🔜
リッチメニューの登録前後出し分け
登録済みユーザーには音声案内メニューに切り替え
🔜
日報送信リマインダー
毎日一定時間に送信催促メッセージを配信
🔜
Cold Start対策（本番移行後に検討）
Cloud Runの最小インスタンス数を1に設定（月数百円のコストが発生）

今後の拡張予定

優先度機能概要
高 Slack本番化 Webhook URL取得・Cloud Run環境変数設定・Apps Scriptプロパティ設定
高 アカウント名・アイコン整備 LINE Official Account Managerで設定
中 リッチメニュー出し分け 登録前：登録案内 / 登録後：音声ガイド・よくある質問
中 日報送信リマインダー 毎日夕方に送信催促を自動配信。Cloud Schedulerで実装予定
低 複数メンバー対応の安定化 同時リクエスト時の状態管理をRedis等に移行
低 Cold Start対策 本番移行・利用頻度次第でmin-instances=1に設定
低 写真・PDF添付 現場写真を日報に添付する機能（要件定義中）
